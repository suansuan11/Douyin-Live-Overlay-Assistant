# 项目名称
Douyin Live Overlay Assistant

# 项目目标
开发一个本地桌面悬浮窗工具，用于在主播直播时实时显示直播间弹幕和互动数据，并保证悬浮窗在游戏之上显示时默认不拦截鼠标事件、不抢前台焦点、不影响游戏操作。

# 一期核心需求
1. 本地桌面端运行
2. 悬浮窗透明背景、圆角、可拖拽、可缩放
3. 默认开启点击穿透
4. 默认置顶显示
5. 默认不抢焦点
6. 支持全局快捷键：
   - Ctrl+Alt+O：显示/隐藏悬浮窗
   - Ctrl+Alt+L：切换点击穿透
   - Ctrl+Alt+Up / Down：调整透明度
   - Ctrl+Alt+1 / 2 / 3：切换布局
7. 实时显示数据事件：
   - 评论弹幕
   - 点赞事件
   - 礼物事件
   - 进入直播间事件（如数据源可提供）
   - 关注事件（如数据源可提供）
8. 支持事件过滤和展示策略：
   - 只显示弹幕
   - 只显示礼物
   - 高亮关键词
   - 屏蔽重复刷屏
   - 合并高频点赞
9. 支持本地配置持久化
10. 支持日志记录和断线重连

# 技术架构
系统分为三层：

## 1. 数据接入层
负责接收直播间实时数据。
输入来源优先级：
A. 官方开放平台接入
B. 本地测试 WebSocket Server
C. Mock JSON 回放

统一输出格式：
{
  "eventId": "uuid",
  "type": "comment|gift|like|enter|follow|system",
  "timestamp": 1710000000000,
  "user": {
    "id": "string",
    "nickname": "string",
    "avatar": "string|null",
    "fansLevel": 0
  },
  "payload": {
    "text": "内容",
    "giftName": "礼物名",
    "giftCount": 1,
    "likeCount": 1
  },
  "raw": {}
}

## 2. 本地事件总线层
负责：
- WebSocket 接收
- 队列缓冲
- 去重
- 节流
- 事件聚合
- 状态管理
- 断线重连

推荐：
- Zustand / RxJS / EventEmitter
- reconnect with exponential backoff
- 本地 ring buffer 保存最近 500~2000 条事件

## 3. 显示层（悬浮窗）
目标：
- 透明背景
- 始终置顶
- 点击穿透
- 不抢焦点
- GPU 渲染流畅
- 支持多模板布局

建议模板：
- 弹幕列表模式
- 礼物高亮模式
- 极简角落模式
- 计数器模式（点赞/礼物/在线趋势）

# 平台与实现建议
优先桌面技术：
1. Electron + React + TypeScript
2. 或 Tauri + React + TypeScript（更轻）
如果选 Electron：
- BrowserWindow transparent
- frame false
- alwaysOnTop true
- skipTaskbar true
- setIgnoreMouseEvents(true, { forward: true })
- 通过快捷键动态切换 ignoreMouseEvents
- Windows 下补充 no activate / topmost / toolwindow 样式策略

# 非功能要求
1. CPU 占用低
2. 内存控制在合理范围
3. UI 帧率稳定
4. 断线后自动重连
5. 所有异常写入日志
6. 不因数据风暴导致 UI 卡死

# 一期不做
1. 自动回复
2. 礼物播报语音
3. AI 回复
4. 多平台接入
5. 用户登录系统
6. 云同步

# 验收标准
1. 在 Windows 10/11 游戏全屏窗口化或无边框模式下，悬浮窗可正常显示
2. 鼠标点击游戏时不会触发悬浮窗
3. 实时接收测试 WebSocket 数据延迟 < 500ms
4. 每分钟 1000+ 高频事件情况下 UI 不崩溃
5. 快捷键切换点击穿透稳定
6. 配置重启后仍可恢复
